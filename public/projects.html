Super Robot Wars

GitHub Deployed App


Notes:
This app was built for my final project at the UT Coding Bootcamp with 2 other cohorts. It is a multiplayer tactical role-playing game based off of the Bandai series Super Robot Wars. I was in charge of building the gameplay UI on the front-end and intergrating socket.io so that the various clients playing would remained synced. Gig.ly

GitHub Deployed App


Notes:
Gig.ly was my second group project at the UT Coding Bootcamp. Our group was randomly assigned so delegating tasks and cooperating with my groupmates was crucial and I believe we executed well on these fronts. I was the front-end engineer tasked with building and designing all of the UI for the app as well as integrating Fullcalendar.io. Gig.ly is a scheduling app for employers to schedule and hire for short-term gigs. The app allows users to create an admin account that can schedule gigs and provides a key that employees can use to sign in and schedule gigs for themselves. As soon as an employee submits a "gig-it" request, employers will receive a text notification with employee's contact info and the gig that was scheduled.

Tech Used:
jQuery, Node.js, Express.js, MySQL, Sequelize, Fullcalendar.io, Moment.js, Twilio, Bulma.io

Notes:
This project was built as a Firebase exercise for week seven of The UT Coding Bootcamp. It is a rock, paper, scissors inspired multiplayer game (instructions are explained in ReadMe and on deployed site). Lessons I learned using Firebase is that you never want to build a game engine on the front end and then attempt to sync clients. If a userâ€™s browser is not running the JS code (i.e. their browser window is minimized) then the clients would desync. If I were to recreate this project I would build the game engine on the back end and then use Firebase to sync the clients.
